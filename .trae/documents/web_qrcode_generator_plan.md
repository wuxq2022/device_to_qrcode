# Web二维码生成器实现计划

## 需求分析

创建一个Web浏览器工具，用于生成设备二维码图片：

* 输入字段：`id`、`external_id`、`serial_number`

* 输出：样式与示例图片一致的二维码图片

* 文件命名格式：`{id}_{external_id}_{serial_number}.png`（如：`173_SN103_D173.png`）

## 示例图片样式分析

根据 `images/173_SN103_D173.png`：

* 白色背景

* 中央放置二维码

* 二维码下方显示文字标签，格式为：`{external_id} ({设备描述}) ({serial_number})`
  示例：`SN103 (全自动抗折抗压试验机) (DYE-300)`

* 整体尺寸约为正方形

## 技术方案

采用**纯前端方案**（无需后端服务器）：

* HTML5 + CSS3 + JavaScript

* 使用 `qrcode-generator` 或 `qrcodejs` 库生成二维码

* 使用 Canvas API 绘制最终图片（二维码 + 文字标签）

* 支持直接下载生成的图片

## 实现步骤

### 步骤1：创建项目结构

```
device_to_qrcode/
├── images/                    # 现有图片目录
├── index.html                 # 主页面
├── css/
│   └── style.css             # 样式文件
└── js/
    └── app.js                # 主逻辑
```

### 步骤2：创建HTML页面 (index.html)

* 添加四个输入框：id、external\_id、serial\_number、device\_description

* 添加"生成二维码"按钮

* 添加预览区域

* 添加"下载图片"按钮

* 引入二维码生成库（CDN）

### 步骤3：创建样式文件 (css/style.css)

* 简洁现代的表单样式

* 响应式布局

* 预览区域样式

* 按钮样式

### 步骤4：实现核心逻辑 (js/app.js)

1. **生成二维码**

   * 使用 QRCode 库生成二维码

   * 二维码内容为 `serial_number`

2. **绘制最终图片**

   * 创建 Canvas

   * 绘制白色背景

   * 绘制二维码到中央

   * 在二维码下方绘制文字标签，格式为：`{external_id} ({device_description}) ({serial_number})`

3. **下载功能**

   * 将 Canvas 转换为 PNG

   * 使用 `{id}_{external_id}_{serial_number}.png` 命名

   * 触发浏览器下载

### 步骤5：测试验证

* 测试生成功能

* 验证图片样式与示例一致

* 验证下载文件名格式正确

## 详细技术参数

### 图片尺寸参数

* 输出图片尺寸：300x350 像素（二维码区域 300x300 + 文字区域 50像素）

* 二维码模块大小：约 10-12 像素/模块

* 边距（quiet zone）：4个模块

### 二维码参数

* 类型：QR Code

* 容错级别：M（Medium，约15%纠错能力）

* 编码模式：Byte（支持字母数字混合）

* 内容：`serial_number` 字段值

### 文字标签参数

* 字体：Arial 或 sans-serif

* 字号：14-16px

* 颜色：黑色 (#000000)

* 对齐：居中对齐

* 位置：二维码下方，居中显示

* 格式：`{external_id} ({device_description}) ({serial_number})`

* 单行显示，如果文字过长则缩小字号或压缩间距
